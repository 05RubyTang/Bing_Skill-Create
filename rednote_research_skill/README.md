## 🚀 快速开始

**方式一：直接说话触发（推荐）**

对 Agent 说：
> "帮我调研一下小红书上的「咖啡赛道」"
> "小红书调研：护肤品平替"
> "研究一下小红书上的露营装备内容"

Agent 会自动：打开浏览器 → 引导扫码 → 搜索 → 采集数据 → 生成报告

---

**方式二：手动指定完整参数**

> "调研关键词「手工皮具」，代表性笔记点赞 >3万，创作者粉丝 >5万，生成报告"

---

## ⚙️ 执行流程（5 步）

```
Step 1 扫码登录      → 浏览器打开小红书，用户扫码，解锁搜索权限
Step 2 关键词搜索    → 多维度关键词搜索，热度排序抓取高赞内容
Step 3 博主验证      → 进入笔记页获取作者 uid，访问主页核验粉丝数
Step 4 数据整合      → 整理真实数据为 JSON，AI 补全洞察分析
Step 5 生成报告      → 渲染为 HTML，存入 research_claw/ 目录
```

详细操作步骤见 **SKILL.md**。

## 📋 报告包含模块

| 模块 | 说明 |
|------|------|
| 数据概览 | 笔记总量、最高点赞、热度级别、核心人群等 |
| 关键词热度 | 子话题热度条形图（★ 评级）|
| 赛道背景 | 趋势分析、风格变化、爆款要素 |
| TOP 笔记 | 真实高赞笔记卡片（含可点击链接）|
| 创作者画像 | 分类博主（含粉丝数、主页链接、典型选题）|
| 用户声音 | 真实评论摘录 |
| 需求与痛点 | 用户想要的 vs 痛点对比 |
| 核心洞察 | 3-5 条可执行的内容洞察 |

## ✅ 质量标准（每次调研必须满足）

- [ ] 代表性笔记点赞数 ≥ 1万（推荐 >3万）
- [ ] 活跃创作者每种类型 ≥ 2 个真实博主
- [ ] 所有笔记链接和博主主页链接真实可访问
- [ ] 粉丝数通过浏览器访问主页实时核验（非估算）
- [ ] HTML 报告纯原生，无第三方 CDN 依赖
- [ ] 数据说明注明为 AI 综合估算，非精确统计

---

## 🔑 技术要点

- **浏览器目标 ID**：每次调研保持同一 targetId，用于搜索+数据抓取
- **笔记链接格式**：`https://www.xiaohongshu.com/explore/{24位16进制ID}`
- **博主主页格式**：`https://www.xiaohongshu.com/user/profile/{uid}`
- **粉丝数提取 JS**：见 SKILL.md Step 3
