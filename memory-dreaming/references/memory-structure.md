# MEMORY.md 三层结构参考

## 🌲 常青层（Evergreen）
**定义**：长期有效，不因时间流逝而失效的信息。
**内容**：Ruby 的偏好和风格、核心工作机制、选题方法论、已踩坑方向。
**更新规则**：只在 Ruby 明确表达了新偏好/新规则时更新；不因「最近没用到」而删除。
**对应 Dreaming 概念**：不受时间衰减影响的常青文件。

## 📅 近期层（Recent）
**定义**：当前有价值但可能随时间过时的信息。
**内容**：当前产品思考主线、近期重要认知（AI动态/研究结论）、近期完成项目索引。
**更新规则**：月度回顾——值得长期记住的升级为常青；已过时/不再相关的清除。
**对应 Dreaming 概念**：时间衰减适用，半衰期约 30 天。

## ♻️ 工具层（Operational）
**定义**：随环境变化需及时更新的实操信息。
**内容**：CLI 命令和已知 Bug、路径配置、版本信息、去重记录位置。
**更新规则**：发现变化随时更新；发现错误立即修正；不保留已失效的工具记录。
**对应 Dreaming 概念**：相当于「用户明确修改偏好时旧记录标废」。

## 评分参考（deep phase 权重对应）

| 信号 | 权重 | 我的轻量实现 |
|------|------|------------|
| 相关性 | 0.30 | 关键词密度（Ruby/决定/确认/教训等） |
| 频率 | 0.24 | 跨多天日志重复出现的主题 |
| 查询多样性 | 0.15 | 被多个不同场景引用的信息 |
| 时效性 | 0.15 | 日志日期距今天数（越近权重越高） |
| 多日巩固 | 0.10 | 连续多天都有记录的主题 |
| 概念丰富度 | 0.06 | 涉及多个层面的深度内容 |

## 提炼判断原则

**应该提炼到常青层的**：
- Ruby 明确说出了偏好/规则（「Ruby 亲口说的」「Ruby 确认的」）
- 选题教训（踩坑了，下次不再重复）
- 工作机制变更（新增了自动任务、文件命名规则等）

**应该提炼到近期层的**：
- 当前产品思考主线和关键结论
- 本周/本月的重要研究发现
- 重要项目的完成记录（附文件路径）

**应该放进工具层的**：
- 新发现的 CLI Bug 和规避方案
- 工具路径变更
- 版本信息更新

**不需要提炼的**（日志里有但不值得长期记住）：
- 具体的小红书笔记数据（已存入 HTML 文件，不需要进 MEMORY.md）
- 一次性的技术操作记录
- 已完成且不会复用的任务细节
